# 首页模块功能说明文档

## 功能概述

首页模块是资金池管理系统的核心展示与监控入口，通过整合多渠道资金池数据与预警信息，为管理员提供直观的数据总览与实时监控能力。该模块汇集了2026年资金池总额、各渠道（苏周到、市民卡、宁波银行）的额度使用情况，以及资金池预警记录列表，帮助管理员快速掌握整体资金运营状态，及时响应异常情况。

## 操作说明

### 2.1 总览数据查看

首页顶部区域展示系统核心数据指标，管理员可在此快速了解资金池整体运营状况。

2026年资金池总额展示区域

该区域位于页面左上方，以醒目的金额格式展示系统当前年度的资金池总额。具体数据项包括：

| 数据项 | 说明 |
|--------|------|
| 资金池总额 | 以"￥X万"格式显示当前年度可用的资金池总金额 |
| 已用额度 | 显示已使用的资金额度（单位：万元）及占比百分比 |
| 剩余额度 | 显示剩余可用资金额度（单位：万元）及占比百分比 |

管理员可通过查看上述三项数据的对比关系，快速判断当前资金池的使用进度。若已用额度接近或超过预设阈值，系统将通过预警机制进行提醒。

### 2.2 渠道资金池监控

页面中部以卡片形式展示三个主要渠道的资金池数据，便于管理员按渠道进行针对性监控。

渠道资金池卡片

三个渠道资金池卡片（苏周到、市民卡、宁波银行）采用统一的布局结构，每个卡片包含以下数据展示：

| 数据项 | 说明 |
|--------|------|
| 总额度 | 当前渠道的资金池总金额（单位：万） |
| 已用额度 | 当前渠道已使用金额，显示数值与百分比（格式：X万 / Y%） |
| 剩余额度 | 当前渠道剩余可用金额，显示数值与百分比（格式：X万 / Y%） |

各渠道卡片的数据相互独立，管理员可通过横向对比不同渠道的资金使用情况，识别使用异常或分配不均的渠道。数据更新频率与系统数据同步机制保持一致，确保监控数据的时效性。

### 2.3 预警记录查询

页面底部展示资金池预警记录列表，管理员可在此查看历史预警信息并进行基础的分页查询操作。

预警记录列表

列表采用标准表格形式展示，包含以下字段：

| 字段名 | 说明 | 数据格式 |
|--------|------|----------|
| 序号 | 预警记录的顺序编号 | 数字 |
| 类型 | 触发预警的渠道名称 | 文本（如：苏周到） |
| 预警类型 | 预警等级（一级/二级/三级） | 文本 |
| 当前使用额度(万元) | 触发预警时的资金使用量 | 数值（保留6位小数） |
| 预警时间 | 预警触发的时间点 | 日期时间（格式：YYYY-MM-DD HH:mm:ss） |

分页查询功能

预警列表底部提供完整的分页控件，支持以下操作：

- 页码导航：通过点击页码数字（1、2、3、4、5）切换至目标页面
- 每页显示条数设置：默认显示10条/页，可根据需要调整
- 快速跳转：通过"跳至页"输入框直接输入目标页码并确认跳转
- 总记录统计：显示"全部 X 条"表明当前查询结果的总数量

管理员可结合实际需要，通过分页功能浏览完整的预警记录历史。预警记录按预警时间倒序排列，最新预警记录显示在列表最前方。

## 注意事项

- 数据时效性：首页展示的数据基于系统定时同步机制，管理员应理解数据存在一定的更新延迟。对于需要实时数据的场景，建议结合对账清单查询模块进行核实。
- 预警响应机制：当预警记录显示某渠道使用额度接近或超过阈值时，管理员应及时进入"主题资金池管理"模块调整额度配置，或在"预警记录查询"模块查看完整预警历史进行分析。
- 权限说明：首页为只读展示模块，不涉及数据修改操作。管理员如需调整资金池配置，需通过相应的管理模块进行操作。
- 分页操作规范：使用"跳至页"功能时，请确保输入的页码在有效范围内（1至总页数），避免因输入错误导致查询异常。
- 数据格式说明：当前展示数据为演示环境数据，实际使用中将以系统真实运行数据填充。部分数值保留多位小数位是为了精确展示预警触发时的实际额度状态。

# 场馆管理模块功能说明文档

## 功能概述

场馆管理模块是后台系统中用于管理场馆信息的核心功能模块，支持对场馆的新增、编辑、查询、停用、删除等全生命周期管理操作。该模块通过多维度筛选条件和分页机制，帮助管理员高效地维护场馆数据，确保场馆信息的准确性和时效性。

## 操作说明

### 1. 场馆查询

场馆查询功能支持多条件组合筛选，帮助用户快速定位目标场馆。

操作步骤：

1. 在查询区域输入或选择筛选条件。可选的筛选条件包括：场馆编号（支持精确匹配）、场馆名称（支持模糊搜索）、公司名称（下拉选择所属公司）、所在区域（下拉选择行政区划）、详细地址（支持模糊搜索）、状态下拉选择启用或停用状态。

2. 点击"查询"按钮，系统根据设置的条件进行筛选，并在下方表格中显示符合条件的场馆记录。

3. 如需清除已设置的筛选条件，点击"重置"按钮即可恢复默认状态，所有查询条件将自动清空。

说明： 多个筛选条件之间为"与"关系，即同时满足所有已设条件的记录才会被检索出来。

### 2. 场馆新增

新增功能用于创建新的场馆记录，扩展场馆资源库。

操作步骤：

1. 在场馆列表页面顶部点击"新增"按钮。

2. 系统弹出新增场馆的表单页面，用户需按照页面提示填写场馆相关信息，包括场馆编号、场馆名称、行政区域、详细地址、所属公司、状态等必填字段。

3. 完成信息填写后，点击确认按钮提交数据，系统将自动校验数据的完整性和格式正确性，校验通过后新场馆将被添加至数据库。

4. 新增成功后，页面自动刷新，新场馆将显示在列表中。

### 3. 场馆编辑

编辑功能用于修改已有场馆的详细信息，确保数据及时更新。

操作步骤：

1. 在目标场馆所在行的"操作"列中，点击"编辑"按钮。

2. 系统弹出编辑表单，加载该场馆的现有数据。

3. 用户可修改除场馆编号外的其他信息字段。场馆编号作为唯一标识，通常不支持修改。

4. 修改完成后，点击确认按钮提交更改，系统将更新数据库中的对应记录。

5. 编辑成功后，列表页面将显示更新后的场馆信息。

### 4. 场馆停用

停用功能用于暂时禁用某个场馆，使其不再对外提供服务或显示，但保留其历史数据。

操作步骤：

1. 在目标场馆所在行的"操作"列中，点击"停用"按钮。

2. 系统弹出确认提示，询问用户是否确定停用该场馆。

3. 点击确认后，场馆状态将变更为"停用"，在列表中的状态列显示为停用状态。

4. 停用后的场馆在常规查询中默认不再显示（除非筛选条件中明确选择"停用"状态），如有需要可随时重新启用。

### 5. 场馆删除

删除功能用于移除不再需要的场馆记录，此操作不可逆，请谨慎使用。

操作步骤：

1. 在目标场馆所在行的"操作"列中，点击"删除"按钮。

2. 系统弹出确认提示，明确告知删除操作将无法恢复，并要求用户二次确认。

3. 确认删除后，系统将从数据库中永久移除该场馆的所有信息。

4. 删除成功后，该场馆将立即从列表中消失。

注意： 建议在删除前确认该场馆下无关联的业务数据，如订单、用户记录等，避免数据异常。

### 6. 分页浏览

分页功能用于管理大规模场馆数据的展示，提升页面加载性能和用户体验。

操作说明：

1. 列表底部显示当前数据总数（如"全部 330 条"）和分页信息。

2. 系统默认每页显示 10 条记录，用户可通过下拉选择每页显示的条数。

3. 点击页码数字（如 1、2、3...33）可跳转至对应页面。

4. 点击"跳至页"输入框并输入目标页码，按回车或点击确认即可快速跳转。

5. 分页状态会记住用户的筛选条件，切换页面时保持当前查询条件不变。

### 7. 修改密码

修改密码功能用于保障用户账户安全，允许用户定期更换登录密码。

操作步骤：

1. 在页面顶部导航或用户菜单中找到"修改密码"功能入口。

2. 系统弹出密码修改表单，包含三个输入字段：旧密码、新密码、验证密码。

3. 按照页面提示输入相应信息：先输入当前使用的密码以验证身份，然后输入新密码并再次输入进行确认。

4. 点击"确定"按钮提交修改请求。

5. 系统将验证旧密码的正确性，以及新密码与验证密码的一致性，验证通过后密码更新成功。

密码安全要求： 新密码通常需要满足一定的复杂度要求（如包含字母、数字、特殊字符等），具体规则以系统实际设置为准。

## 注意事项

1. 数据完整性： 在新增或编辑场馆时，请确保必填字段（如场馆名称、所属公司、行政区域等）已正确填写，否则系统将提示无法提交。

2. 编号唯一性： 场馆编号在系统中具有唯一性标识作用，新增时系统会自动检测编号是否重复，请勿手动设置与现有场馆相同的编号。

3. 删除谨慎性： 删除操作将永久移除数据且无法恢复，请在执行删除前确认该场馆确实不再需要，建议优先使用停用功能作为替代方案。

4. 状态管理： 场馆状态直接影响其对外展示和业务运行，停用操作会使得场馆在用户端不可见，如有正在进行的业务订单，请妥善处理后再变更状态。

5. 筛选逻辑： 使用筛选功能时，多个条件之间为逻辑"与"关系，如筛选结果为空，请适当减少筛选条件或检查条件设置是否正确。

6. 分页数据一致性： 在执行新增、编辑、删除操作后，如当前页已无数据，系统会自动跳转至前一页或首页，确保列表始终显示有效数据。

7. 密码安全： 修改密码时，建议设置与其他平台不同的密码，并定期更换。切勿将密码告知他人或在公共环境下输入密码。

# 主题资金池管理模块功能说明文档

## 功能概述

主题资金池管理模块是后台系统的核心资金管控模块，用于统一管理各类应用渠道（如苏周到、市民卡、宁波银行等）的年度资金预算额度。该模块支持资金额度配置、预警阈值设置、额度注入与减额操作，以及用户级别的折扣和限额管理，同时完整记录所有额度变更历史，确保资金使用的可追溯性和合规性。通过多级预警机制和变更记录功能，帮助管理员实时监控资金池状态，有效防范资金风险。

## 操作说明

### 1. 年份管理

#### 1.1 切换年份查看

在页面顶部年份下拉菜单中选择目标年份（当前示例为2025年），系统将自动加载该年份下的所有资金池数据、用户额度配置及变更记录。年份切换后，主题资金池表格、用户额度表格和变更记录表格中的数据将同步更新为所选年份的对应信息。

#### 1.2 新增年份

点击"新增年份"按钮可创建新的资金池管理年度。系统将跳转到年份新增页面，管理员需设置新年度的初始参数，包括各渠道的初始额度、预警阈值、时间范围等配置。新年份创建完成后，将自动生成该年度的完整资金池体系，可进行后续的额度调整和预警设置操作。

### 2. 主题资金池管理

#### 2.1 查看资金池列表

主题资金池表格展示了指定年份下各渠道资金池的完整配置信息，包含以下数据列：年份、类型（应用渠道名称）、总额度、一级预警值、二级预警值、三级预警值、开始时间、结束时间以及当前状态。表格支持滚动查看更多数据，每页默认显示10条记录，可通过底部翻页控件进行页面跳转或调整每页显示条数。

#### 2.2 预警阈值设置

点击操作列中的"预警设置"按钮，弹出预警设置弹窗。管理员可为对应的资金池配置三级预警阈值：一级预警值（最低阈值）、二级预警值（中间阈值）、三级预警值（最高阈值）。当资金池剩余额度触及各层级预警线时，系统将根据预设规则触发相应的预警通知。三级预警值应大于二级预警值，二级预警值应大于一级预警值，且各级预警值均不得超过资金池总额度。

#### 2.3 启用/停用资金池

在操作列中点击"启用"或"停用"按钮，可切换资金池的当前状态。启用状态的资金池可正常进行资金注入、消费抵扣等业务操作；停用状态的资金池将暂停所有资金业务，仅保留查询和配置功能。状态切换成功后，系统将自动更新变更记录，记录操作人员、操作时间及状态变更信息。建议在重要业务调整或系统维护时合理使用启用/停用功能。

#### 2.4 资金注入

点击操作列中的"注入"按钮，弹出资金注入对话框。管理员需输入注入金额（单位：万元），确认后系统将把相应金额添加至目标资金池的总额度中。注入操作不可撤销，请务必核对注入金额的准确性。资金注入后，系统将自动更新该资金池的当前总额度，并生成一条变更记录，包含修改前总额、修改后总额、操作人及操作时间等信息。

#### 2.5 资金减额

点击操作列中的"减额"按钮，弹出资金减额对话框。管理员需输入减额金额（单位：万元），确认后系统将从目标资金池的总额度中扣除相应金额。减额操作不可撤销，且减额后的剩余额度不得为负数。减额操作适用于资金回收、业务调整等场景，操作完成后系统将自动记录变更历史，便于后续审计和追溯。

### 3. 用户额度管理

#### 3.1 查看用户额度配置

用户额度表格展示了当前选中年份下的用户级别额度配置参数，包括：年份、折扣比例、单笔限额（单笔消费最高金额）、折上折折扣额度（叠加优惠后的最高优惠金额）、结束时间以及当前状态。管理员可通过此表格快速了解用户的消费限制和优惠策略配置情况。

#### 3.2 设置用户额度参数

点击操作列中的"设置"按钮，弹出用户额度设置对话框。管理员可分别修改以下参数：折扣比例（范围0-1，表示用户消费时可享受的折扣率）、单笔限额（单笔交易的最高金额上限）、折上折折扣额度（在折扣基础上再享受优惠时的最高优惠金额）、结束时间（配置生效的截止日期）。各项参数设置完成后点击确定保存，用户额度配置将立即生效并记录变更历史。

### 4. 变更记录查询

#### 4.1 查看变更历史

变更记录表格完整保存了所有资金池额度的变更流水，按操作时间倒序展示。表格包含以下信息：序号（变更记录编号）、类型（操作的渠道类型）、修改前总额（万元）、修改后总额（万元）、修改人姓名（操作人员账号）、修改时间（操作发生的具体时间）。通过变更记录，管理员可追溯任意时间点的资金池状态变化，支持财务审计和异常排查。

#### 4.2 变更记录分页

变更记录表格底部提供分页控件，显示总记录条数（示例为14条）、当前页码、每页显示条数选择器（默认10条/页）以及页码跳转输入框。管理员可根据需要调整每页显示数量或直接输入页码跳转到指定页面，快速定位目标变更记录。

### 5. 密码管理（全局功能）

点击页面顶部导航栏中的"修改密码"选项，弹出密码修改对话框。该功能为系统全局功能，需填写以下信息：旧密码（当前登录账号的密码）、新密码（需要设置的新密码）、验证密码（再次确认新密码）。点击"确定"提交修改请求。新密码必须与旧密码不同，且需符合系统密码复杂度要求（通常包含大小写字母、数字及特殊字符）。密码修改成功后，系统将要求使用新密码重新登录。

## 注意事项

在操作主题资金池管理模块时，管理员应特别注意以下事项，以确保资金安全和管理规范。

密码修改功能需验证旧密码，请确保在安全的网络环境下操作，不要将新密码透露给他人，定期更换密码以保障账号安全。

# 预警记录查询模块功能说明文档

## 功能概述

预警记录查询模块用于查询和管理系统中产生的各类预警记录。通过该模块，管理员可以查看不同类型和等级的预警事件，了解当前使用额度情况及预警发生时间，支持多维度筛选和分页浏览，帮助运营人员及时掌握资金池预警状态并采取相应措施。

## 操作说明

### 1. 访问模块

在系统左侧导航栏中点击"预警记录查询"菜单项，进入预警记录查询列表页面。页面路径显示为"主页/预警记录查询/预警记录列表"。

### 2. 查询预警记录

系统提供多条件组合查询功能，具体操作步骤如下：

关键字查询：在"关键字"输入框中输入要搜索的关键词，支持按类型名称等字段进行模糊匹配查询。

类型筛选：点击"类型"下拉框，选择特定类型进行筛选。如不选择则默认查询全部类型记录。

执行查询：设置完查询条件后，点击"查询"按钮，系统根据设定条件返回符合条件的预警记录列表。

重置条件：如需清空已设置的查询条件并重新查询，点击"重置"按钮恢复默认状态。

### 3. 查看预警详情

预警记录列表以表格形式展示各预警记录信息，包含以下字段：

| 字段名称 | 说明 |
|---------|------|
| 序号 | 记录的唯一编号标识 |
| 类型 | 预警所属的业务类型，如"苏周到" |
| 预警类型 | 预警等级，分为一级、二级、三级 |
| 当前使用额度(万元) | 触发预警时的资金使用额度，单位为万元 |
| 预警时间 | 预警发生的时间，格式为yyyy-MM-dd HH:mm:ss |

在字典列表中找到需要修改的记录，点击该行记录末尾的"修改"操作链接。系统将弹出编辑窗口，显示当前字典的详细信息。用户可对字典类型名称、字典名称和字典值进行修改。确认修改内容无误后，点击"确定"提交更改，系统将更新对应记录并刷新列表显示。

### 4. 删除字典数据

在字典列表中找到目标记录，点击该行记录末尾的"删除"操作链接。系统将弹出确认提示，询问用户是否确定删除该字典数据。点击"确定"执行删除操作，记录将从系统中移除；点击"取消"则放弃删除操作。注意：删除操作不可恢复，关联业务数据可能受影响，请谨慎操作。

### 5. 密码修改

用户可通过页面顶部的导航栏或用户菜单进入"修改密码"功能。系统将弹出密码修改窗口，用户需依次输入旧密码、新密码和验证密码。为确保账户安全，新密码应与旧密码不同，且验证密码必须与新密码完全一致。完成信息填写后，点击"确定"提交修改请求。系统将验证旧密码的正确性，并在验证通过后更新用户密码。

## 注意事项

字典数据作为系统基础数据，被多个业务模块引用共享。在进行字典数据的修改或删除操作前，请确认该字典项未被其他业务数据引用，避免造成数据关联错误或业务功能异常。建议在非业务高峰期进行字典维护操作，并做好数据变更记录。

新增字典数据时，请确保字典类型名称、字典名称和字典值的命名规范统一，便于后续数据管理和维护。字典值应保持唯一性，避免重复值导致业务逻辑混乱。

密码修改时，请确保新密码符合系统安全策略要求，建议使用包含字母、数字和特殊字符的组合密码，并定期更换以保障账户安全。修改成功后，系统可能要求用户重新登录以应用新密码。
